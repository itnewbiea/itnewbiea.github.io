CentOS7 中MySQL的安装以及MySQL密码的修改

         在CentOS系统中会默认安装有MariaDB，可能根据需要我们要安装MySQL，在MySQL安装完毕后会直接替换到系统中自带的MariaDB，以下操作均在Terminal中操作：

1、首先下载并安装MySQL官方需要的 yum repository

1.1  先使用下面的命令切换到 root 用户

[parallels@centos-linux-7 ~]$ sudo su
[sudo] password for parallels:  自己电脑的开机密码
[root@centos-linux-7 parallels]#
[root@centos-linux-7 parallels]# sudo wget -i -c http://dev.mysql.com/get/mysql57-community-release-el7-10.noarch.rpm

执行命令后的画面：

2、直接用下载好的 yum 进行安装

[root@centos-linux-7 parallels]# sudo yum -y install mysql57-community-release-el7-10.noarch.rpm

执行命令后的画面：

3、直接用 yum 安装 MySQL 服务器

[root@centos-linux-7 parallels]# sudo yum -y install mysql-community-server

执行命令后的画面，这一步可能需要点时间需要我们耐心等待下，网速快的话很快就会好，而且在这一步我们会发现我们安装的 MySQL 会 替换 掉原来的 MariaDB，如果不出意外一切会很顺利 

4、启动 MySQL 服务器

[root@centos-linux-7 parallels]# systemctl start mysqld.service

这一步我们是看不到画面，但是我们可以使用下面的命令来查看 MySQL 服务器的运行状态

看到上面画面的中两个红色框框中的内容，我们会发现 MySQL 服务器已经正常启动，卧槽，这么简单

5、接下来我们要查看进入 MySQL 的密码了：

[root@centos-linux-7 parallels]# sudo grep "password" /var/log/mysqld.log

执行命令后的画面，红色框框中的内容就是进入 MySQL 大门的  “金钥匙”

6、接下来我们就可输入下面的命令进入 MySQL 了

[root@centos-linux-7 parallels]# mysql -uroot -p

    6.1  紧接着我们要输入刚开看到红色框框中的那个 “金钥匙” 了，可能是因为我手残的原因输入了好几遍 “金钥匙” 都进不去，卧槽，阿西吧，什么东东啊 ，画面如下

     6.2  用其他的方法，我们可以尝试着修改他的配置文件试试吗，接下来我们切换到配置文件所在的目录：

[root@centos-linux-7 parallels]# cd /etc

我们发现有我们需要的那个文件了,红色框框中的那个就是了，确认过眼神，它就是我们需要文件了  :

使用一下命令打开，然后按 “i” 键，进入编辑，然后再文件末尾添加 skip-grant-tables ,意思就是在进入 MySQL 时无需输入密码直接进入就可以

[root@centos-linux-7 etc]# sudo vim my.cnf

执行命令和添加代码后的画面

将 skip-grant-tables 添加完成后直接按 “ESC” 键退出编辑，然后按快捷键 “shift + zz” (此处注意 “z” 要快速连按 2 次哈) 保存并退出或者输入“:”接着输入 “wq” 保存并退出也可以

7、直接输入下面两行命令：

停止 MySQL 服务器：  systemctl stop mysqld.service
启动 MySQL 服务器：  systemctl start mysqld.service

8、再次输入进入 MySQL 的命令，然后在要求输入密码的时候直接按回车键就可以，你会发现 卧槽，居然不用密码也能进入，但是出于安全考虑我们还是要设置密码的哦

[root@centos-linux-7 parallels]# mysql -uroot -p

9、进入 MySQL 后，输入一下命令进行权限的刷新：

mysql> FLUSH PRIVILEGES;

接下来，再次输入如下命令修改密码即可：

mysql> AlTER USER 'root'@'localhost' IDENTIFIED BY '123456';

10、我们再次进入那个配置文件中，将添加的代码注释掉或者删掉就可以，停止 MySQL 服务器，运行 MySQL 服务器，再次进入  MySQL，直接输入修改后的密码就可以了
