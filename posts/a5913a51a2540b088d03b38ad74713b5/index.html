c# Win Form程序调用Web API

关于在winform中调用webapi

在Win Form中可以使用WebClient、HttpWebRequest、HttpClient中的一个类来调用Web API。由于我自己用的是VS2010，最高只有 .Net FrameWork 4.0，而HttpClient是 .Net FrameWork 4.5里的，所以这里只介绍WebClient和HttpWebRequest的用法。

1、WebClient

初始化一个WebClient类的对象，然后设置client对象的Headers（我这里使用的是json格式的字符串传参，并且编码是utf-8格式的）的Content-Type，最后调用client的UploadString()方法（我这里是要给服务器传入数据并获取返回是否成功，所以调用此方法，各位可根据不同需求调用此类的其他方法），传入url（WebAPI地址）、“Post”（指定改WebAPI是post还是get请求）、sendData（传给WebAPI的数据，此处为json格式的字符串）。以下是使用的代码：

public string RequestWebAPI(string sendData, string url)
{
    try
    {
        string backMsg = "";
        WebClient client = new WebClient();
        client.Headers.Add("Content-Type", "application/json;charset=utf-8");
        backMsg = client.UploadString(url, "POST", sendData);
        return backMsg;
    }
    catch (Exception ex)
    {
         return ex.Message;
    }
}

2、HttpWebRequest

初始化HttpWebRequest对象，由于HttpWebRequest类无法通过new创建，需要使用WebRequest.Create(url)方法初始化（url为WebAPI地址），而后通过设置属性Method指定请求是Post还是get，属性ContentType设置传入的参数格式（此处为utf-8编码的json格式字符串）。将要传入的数据（即json格式字符串sendData）转换为byte字节数组，而后通过请求流写入存于缓冲区的byte字节数组数据。然后获取响应流对象，此处从响应流读取数据有两种方法，一种是通过将响应流中的数据读取到byte字节数组，而后转换为字符串；另一种是通过初始化StreamReader类对象读取，以下为使用代码：

public string RequestWebAPI(string sendData, string url)
{
    try
    {
        string backMsg = "";
        HttpWebRequest httpRquest = (HttpWebRequest)WebRequest.Create(url);
        httpRquest.Method = "POST";
        httpRquest.ContentType = "application/json;charset=utf-8";
        byte[] dataArray = Encoding.UTF8.GetBytes(sendData);
        Stream requestStream = null;
        if (!string.IsNullOrEmpty(sendData))
        {
            requestStream = httpRquest.GetRequestStream();
            requestStream.Write(dataArray, 0, (int)dataArray.Length);
            requestStream.Close();
        }

        HttpWebResponse response = (HttpWebResponse)httpRquest.GetResponse();
        Stream responseStream = response.GetResponseStream();
        #region byte字节数组读取
        byte[] data = new byte[4096];
        int count = responseStream.Read(data, 0, data.Length);
        if (count != 0)
        {
            backMsg = Encoding.UTF8.GetString(data, 0, count);
        }
        #endregion

        #region StreamReader读取

        //StreamReader myStreamReader = new StreamReader(responseStream ,Encoding.GetEncoding("utf-8"));
        //backMsg = myStreamReader.ReadToEnd();
        //myStreamReader.Close();
        //myStreamReader.Dispose();

        #endregion
        requestStream.Dispose();
        responseStream.Close();
        responseStream.Dispose();

        return backMsg;
    }
    catch (Exception ex)
    {
        return ex.Message;
    }
}

3、对于HttpWebRequest类使用中出现的问题解决

在使用HttpWebRequest调用WebAPI的程序写完后，在我自己的电脑上测试成功，但是在我发给别人使用时却出现了读取返回值乱码的问题，而且是别人的电脑都不行，只有我自己的电脑可以。一开始我以为是读取时响应流返回值编码的问题（毕竟WebAPI是别人写的，虽然一般都使用utf-8编码，但不保证所有人都是），但是经过不断测试，发现不管使用什么编码（GBK、GB2312、Unicode，能想到的都试了），结果都是一样的乱码，所以断定这不是编码的问题；而后我在使用我自己的电脑和另外一台出现乱码的电脑一起调试，发现在

HttpWebResponse response = (HttpWebResponse)httpRquest.GetResponse();

这里创建的response对象里的ContentEncoding属性和ContentLength属性两台电脑不一样，在出现乱码的电脑上ContentEncoding为“gzip”，ContentLength为正数，而在我自己的电脑上这两个值分别为空和-1。终于明白原来是接收到的response响应被压缩了，所以读取出来的值会乱码。解决的代码如下：

using (GZipStream gzip = new GZipStream(response.GetResponseStream(), CompressionMode.Decompress))
{
    byte[] data = new byte[4096];
    int count = gzip.Read(data, 0, data.Length);
    if (count != 0)
    {
        backMsg = Encoding.UTF8.GetString(data, 0, count);
    }
    gzip.Close();
    gzip.Dispose();
}

这里使用到的GZipStream需要引用using System.IO.Compression;
