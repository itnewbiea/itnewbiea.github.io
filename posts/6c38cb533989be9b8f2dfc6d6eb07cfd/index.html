linux httpd源码包安装详解 哭了

终于完成，记录一下方便以后查看

一：安装准备

  1. 安装C语言编译器：gcc
    因为源码包为C语言编写，后面操作会对它编译。
    [root@localhost ~]# yum -y install gcc
  2. 下载httpd源码包
    http://mirror.bit.edu.cn/apache/httpd/

二：安装注意事项

  1. 软件保存位置根据自己习惯，但绝不能随意保存，我为/usr/local/apache
  2. 如何确定安装过程有报错：安装过程停止后查看后几行是否有
    Error，Warning，No等关键字

三：源码包安装

  1. 将Windows下载好源码包的用工具传到Linux中。
    [root@localhost ~]# tar -zxvf httpd-2.4.38 [root@localhost ~]# tar -zxvf httpd-2.4.38
  2. 解压后进入解压出的目录中，[root@localhost ~]# cd httpd-2.4.38/(里面的INSTALL文件里面有安装的详细步骤，可以参考)

    进入后先进行软件的配置和检查
    使用./configure命令
    [root@localhost httpd-2.4.38]# ./configure --prefix=/usr/local/apache
    -prefix后要跟位置，系统会把文件安装到一个目录中，若无会默认到多个分散，删除等操作不方便
    这时候，最头疼的来了，报错了！
    报错信息我就不贴了，原因基本一样，没有apr，apr-util，pcre包
    下载！
  • apr, apr-util http://apr.apache.org/download.cgi
  • pcre-http://www.pcre.org/
  1. 三个完成下载后：重复步骤
    解压：tar -zxvf + 包名
    进入目录：cd + 文件名
    软件配置与检查：.configure --prefix= + 位置
    !注意：在apr-util的命令中还会报错,
    它的命令为：.configure --prefix= + 位置 --with-apr=+位置
    在pcre过程中也会报错，需c++
    只需要：yum install -y gcc gcc-c++即可
    在apr，apr-util，pcre包都成功进行    .configure    后
  2. 分别进行 make命令进行编译
    make install来编译安装
    结束后，查看包是否在指定位置
    [root@localhost httpd-2.4.38]# ls /usr/local/
    会发现已经成功了！马上成功了！
  3. 最后进行httpd的软件配置与检查，还是
    .configure命令，不过要加上with选项，来处理报错信息！
    [root@localhost httpd-2.4.38]# ./configure --prefix=/usr/local/apache --with-apr=/usr/local/apr --with-apr-util=/usr/local/apr-util --with-pcre=/usr/local/pcre
    耐心等待你会发现成功了！
    最后收尾工作：
    [root@localhost httpd-2.4.38]# make ——编译
    [root@localhost httpd-2.4.38]# make install ——安装
    结束！

四：启动服务器

在解压包中的INSTALL文件中可以查看启动命令

前三条为安装命令，最后一个为启动命令/bin前为你的安装位置
例如我的为
[root@localhost httpd-2.4.38]# /usr/local/apache/bin/apachectl start
启动后，网页输入ip地址，看见

成功！

补充：编译apr-util是会出现错误信息”apr-util致命错误:expat.h:没有那个文件或目录“
所需文件在expat库中
在make前需要安装
解决方法 ：yum -y install expat-devel
