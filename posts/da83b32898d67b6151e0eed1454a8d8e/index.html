问题描述

微服务调用时，传入的参数为List类型；经过调试，当传入list.size()小于7时，可以正常调用，大于等于7时，服务调用失败，报错显示object is not an instance of declaring class。

解决方法

在调用端服务的配置文件中添加feign的配置，配置如下：

feign:
  compression:
    request:
      # 开启请求压缩
      enabled: false
      # 配置压缩的 MIME TYPE
      mime-types: text/xml,application/xml,application/json
      # 配置压缩数据大小的下限
      min-request-size: 2048
    response:
      # 开启响应压缩
      enabled: true

即将feign的请求压缩关闭，不对数据进行压缩，因为压缩后数据会加上转义字符，导致接收数据时，对象无法正常转换。
