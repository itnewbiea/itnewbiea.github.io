AJAX请求 $.ajaxSetup方法的使用

jQuery.ajaxSetup()函数用于设置AJAX的全局默认设置。

该函数用于更改jQuery中AJAX请求的默认设置选项。之后执行的所有AJAX请求，如果对应的选项参数没有设置，将使用更改后的默认设置。

该函数属于全局jQuery对象。

语法

jQuery 1.1 新增该静态函数。

[html]  view plain  copy
  1. jQuery.ajaxSetup( settings )  

参数

参数描述
settingsObject类型一个对象，其中的每个属性表示需要更改默认设置的选项，属性值表示更改后的默认值。

关于settings参数可以识别的属性，请参考jQuery.ajax()中的settings参数说明。其中的所有选项参数都是可选设置的。

返回值

jQuery.ajaxSetup()函数没有返回值，或者说其返回值为undefined。

示例&说明

以下是与jQuery.ajaxSetup()函数相关的jQuery示例代码，以演示jQuery.ajaxSetup()函数的具体用法：

[html]  view plain  copy
  1. //设置AJAX的全局默认选项  
  2. $.ajaxSetup( {  
  3.     url: "/index.html" , // 默认URL  
  4.     aysnc: false , // 默认同步加载  
  5.     type: "POST" , // 默认使用POST方式  
  6.     data:{
           "name":"limian"  //默认添加额外参数
        }
  7.     headers: { // 默认添加请求头  
  8.         "Author": "CodePlayer" ,  
  9.         "Powered-By": "CodePlayer"  
  10.     } ,  
  11.     error: function(jqXHR, textStatus, errorMsg){ // 出错时默认的处理函数  
  12.         // jqXHR 是经过jQuery封装的XMLHttpRequest对象  
  13.         // textStatus 可能为： null、"timeout"、"error"、"abort"或"parsererror"  
  14.         // errorMsg 可能为： "Not Found"、"Internal Server Error"等  
  15.   
  16.         // 提示形如：发送AJAX请求到"/index.html"时出错[404]：Not Found  
  17.         alert( '发送AJAX请求到"' + this.url + '"时出错[' + jqXHR.status + ']：' + errorMsg );          
  18.     }  
  19. } );  
  20.   
  21.   
  22. // 未设置任何参数，但url、async、type、headers、error、data等参数的默认值均已被$.ajaxSetup()更改(如上)  
  23. $.ajax( );  
  24.   
  25.   
  26. // 设置了url、type、success、error，就使用自己设置的参数值  
  27. // 但async、headers等参数的默认值已被$.ajaxSetup()更改  
  28. $.ajax( {  
  29.     url: "myurl.php?action=list" ,  
  30.     type: "GET" ,  
  31.     success: function( data, textStatus, jqXHR ){  
  32.         alert("返回数据：" + data);  
  33.     } ,  
  34.     error: function(jqXHR, textStatus, errorMsg){  
  35.         alert("自己的error!");          
  36.     }  
  37. });  
  38.   
  39. // 上述$.ajaxSetup()的默认设置对$.get()、$.post()、load()、$.getJSON()、$.getScript()等AJAX函数也生效  
  40. // 因为这些函数也是在内部调用$.ajax()函数来实现的，只不过它们在内部都设置了一些参数  
  41.   
  42.   
  43. // $.get()在内部调用$.ajax()时已设置type为"GET"，因此仍然使用GET方式。  
  44. // 但url、async、headers、error等参数的默认值已被$.ajaxSetup()更改  
  45. $.get( );

  49. // url也被设置为"user/action.php?method=addUser"  
  50. // 但async、headers、error等参数的默认值已被$.ajaxSetup()更改  
  51. $.post( "user/action.php?method=addUser" );
