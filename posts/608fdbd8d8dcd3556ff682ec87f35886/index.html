Android中将APK放入系统APK目录中,启动失败

Android中将APK放入系统APK目录中,启动失败.

背景:有时候我们需要将apk放进系统里面去，拷贝到system/app/或者打包进system.image中。
问题:有时候apk需要相应的so库，这时候直接拷贝进system/app，apk程序无法正常运行，会报错.

问题分析:
我们首先来分析一下原因：
1、同样的包，同样的so，通过install就能正常运行
我们去掉在应用的manifest.xml中添加的android:sharedUserId=“android.uid.system”，然后重新编译打包，使用adb install直接安装，是不是很神奇，能够正常运行。
2、回归主题
我们是要解决在app是系统应用的情况下，也就是manifest.xml中添加的android:sharedUserId=“android.uid.system”，
使用push将包推到/system/app下的情况（这里也要特别说一下，Android5.0之后的Android,最好将apk文件push到/system/priv-app目录）。
步骤如下，编译打包，然后adb push xxx.apk /system/app，重启，来了，崩溃了，逮住日志看一下，从日志中可以看出抛出java.lang.UnsatisfiedLinkError，
从nativeLibraryDirectories=[/vendor/lib, /system/lib]]]: findLibrary returned null
这句话可以看出，它从/vendor/lib, /system/lib这两个路径去找so文件，结果没有找到（findLibrary returned null）。
这又是为什么呀？
原因是/system/app下面的APK成为系统级app时,包含SO文件，app默认加载so库的路径就会变成/system/lib，而system/lib却是只读的，导致so库无法正常解压不会自动安装，
需要手动把so文件push 到 "/system/lib"目录下面。

解决方案:
a. 将apk中的so文件手动push到"/system/lib"
b.思路是将apk中的so库复制到/data/data/应用包名/lib的安装目录中，再通过反射更改app加载so库的路径， 代码下载请点击
c.1.通过adb命令将apk安装到系统中。
adb install C:\Users\ad\app\DemoApp.apk

DemoApp.apk程序会安装到data/app/目录下。

2.通过adb命令将data/app/下面安装目录拷贝出来
adb pull data/app/com.demo.demoapp-jcHXPygyFs5ki11VUK-oBQ==

3.拷贝出来的文件夹中就包含了.so文件

将base.apk改名为DemoApp.apk，文件夹com.demo.demoapp-jcHXPygyFs5ki11VUK-oBQ==改名为DemoApp，
在将整个DemoApp文件夹拷贝到system/app/下面。
adb push C:\Users\ad\app\DemoApp system/app/
d.预置APK到android系统中.需要修改Android.mk文件.
