归并排序

归并排序基于归并这个简单的操作，即将两个有序的数组归并成一个更大的有序数组。要将一个数组排序，可以先递归的将它分成两半分别排序，然后将结果归并起来。

归并排序能保证将任意长度为N的数组排序所需时间和NlogN成正比，它的主要缺点是它所需的额外空间和N成正比。

原地归并的抽象方法

将涉及到的所有元素复制到一个辅助数组中，再把归并的结果放回原数组中

代码：

void MergeSort::merge(std::vector<int>& a, int lo, int mid, int hi)
{
    int i = lo, j = mid + 1;
    for (int k = lo; k <= hi; k++)
        aux[k] = a[k];
    for (int k = lo; k <= hi; k++)
        if (i > mid)
            a[k] = aux[j++];
        else if (j > hi)
            a[k] = aux[i++];
        else if (aux[j] < aux[i])
            a[k] = aux[j++];
        else
            a[k] = aux[i++];
}

自顶向下的归并排序

递归归并是分治思想中最典型的一个例子

代码

void MergeSort::sort(std::vector<int>& a, int lo, int hi)
{
    if (hi <= lo)
        return;
    int mid = lo + (hi - lo) / 2;
    sort(a, lo, mid);
    sort(a, mid + 1, hi);
    merge(a, lo, mid, hi);
}

sort() 方法的作用其实在于安排多次merge()方法调用的正确顺序

对于长度为N的任意数组，自顶向下的归并排序需要 1 / 2 N l g N 1/2NlgN 1/2NlgN 至 N l g N NlgN NlgN 次比较
对于长度为N的任意数组，自顶向下的归并排序最多需要访问数组 6 N l g N 6NlgN 6NlgN 次

使用插入排序处理小规模的子数组一般可以将归并排序的运行时间缩短 10%~15%

自底向上的归并排序

代码量更少

代码：

void MergeSort::MergeBU(std::vector<int>& a)
{
    int N = a.size();
    aux.resize(N);
    for (int sz = 1; sz < N; sz = sz + sz)
        for (int lo = 0; lo < N - sz; lo += sz + sz)
            merge(a, lo, lo + sz - 1, std::min(lo + sz + sz - 1, N - 1));
}

子数组的大小初始值为1，然后每次加倍

对于长度为N的任意数组，自底向上的归并排序需要 1 / 2 l g N 1/2lgN 1/2lgN 至 N l g N NlgN NlgN 次比较，最多需要访问数组 6 N l g N 6NlgN 6NlgN 次

当数组长度为2的幂时，自顶向下和自底向上的归并排序所用的比较次数和数组访问次数正好相同，只是顺序不同。自底向上的归并排序比较适合用链表组织的数据。

没有任何基于比较的算法能够保证使用少于 l g ( N ! ) lg(N!) lg(N!) ~ N l g N NlgN NlgN 次比较将长度为N的数组排序

归并排序是一种渐进最优的基于比较排序的算法。

归并排序和希尔排序的运行时间之间的差距在常数级别内。
